# KTB 1주차 프로젝트 회고

---

1주차 프로젝트인 만큼 간단한 CLI 프로젝트라고 생각하고 편하게 완성할 수 있을거라고 생각한 내가 부끄러웠다.

물론 내가 AI 마냥 뚝딱 완성할 수 없는 건 당연하겠지만 예상보다 더 고생한 것 같다.

C언어를 주로 공부하고 프로젝트를 했던 지라 자바라는 언어로 1주차 프로젝트를 하는데에 있어서 굉장히 어려움을 겪었다.

나름 코테를 준비하며 어느 정도 자바에 익숙해져 있다고 생각했었는데, 나의 오만한 착각이었다.

프로젝트를 시작하며 내가 세운 목표는 자바라는 언어를 사용하는 만큼 내 능력 안에서 최대한 객체 지향 특성을 살려 구현하는 것이였다.

---

### 잘했다고 생각하는 점

1. 클래스 다이어그램에 대해 구글링 및 공부를 하면서, 무작정 프로젝트 코드 구현을 시작한 것이 아니라 필요한 클래스들과 기능들을 설계하고, 그들간의 관계와 성격에 대해 손으로 직접 설계도를 작성하고 시작하였다.

2. 자바로는 코테만 준비해봤는데, 그래도 이론적으로 알고 있는 객체 지향 프로그래밍을 적용했다고 생각한다.
(예를 들어 값을 강제해서 안정성과 가독성을 높여주는 열거형 사용을 시도해본 점, 그러나 적절한 경우에 사용했는지는 확신이 없다...)

---

### 어려웠던 점과 아쉬운 점

1. 변수나 메서드를 선언할 때 느낌으로 접근 제어자를 선언하는 것 같다. 확신을 가지고 조금 더 정답에 가까운 선택을 하고 싶다.
1주차 프로젝트를 진행하면서는 외부에서 접근 가능하여 값이 변경되면 안될 거 같은데 ...? 해서 private를 선언했다.


2. 회고를 작성하며 내가 짠 코드를 다시 보니 하드 코딩이 몇몇 있는 거 같다. 2주차 프로젝트부터는 더 많이 고민해서 하드 코딩을 최대한 줄여봐야겠다.
(ex, readInt(), readMenuId(), readQty(), readForkCount() 의 경우, readInt(min, max) 를 통해서 공통 메서드로 만드는 편이 훨씬 좋아보인다.)


3. 곳곳에 예외처리도 미흡한 거 같다. 좋은 코드의 기본이 예외 처리라고 이론으로 배웠는데 막상 내가 짠 코드를 보니 예외 처리가 필요한 곳이 추가로 보인다. 추후에는 더 신경쓸 수 있도록 해야겠다.

---

### 최종 느낀점

프로그래밍 언어와 상관 없이 코드 자체를 구현하는 능력이 아직 많이 부족하다고 생각이 들었고, 자바를 더 많이 공부하고 직접 코드를 짜보며 더 많이 부딪혀 봐야겠다.
